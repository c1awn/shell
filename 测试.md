### 1.linux任务计划cron
1. 计划任务模板
![](http://os9ep64t2.bkt.clouddn.com/17-7-17/38645446.jpg)
其中时间格式特别注意：  
#分范围0-59，时范围0-23，日范围1-31，月范围1-12，周0-6或者1-7  
#可用格式1-5表示一个范围1到5  
#可用格式1,2,3表示1或者2或者3  
#可用格式 */2表示被2整除的数字，比如小时，那就是每隔2小时
2. 实际编辑一条计划任务
```
[root@cs701 ~]#crontab -e #实际编辑一条计划任务
0 3 1-10 */2 2,5 /bin/bash /usr/local/sbin/123.sh >>/tmp/123.log 2>>/tmp/123.log
```
#偶数月周二周五的1-10号3点钟，执行123.sh命令，正确和错误日志输出到123.log  
>要启动这个服务，需要systemctl start crond  

**如果要检查是否启动服务，有2种命令：**
```
[root@cs701 ~]#ps aux |grep cron
root        566  0.0  0.1 127876  1680 ?        Ss   19:45   0:01 /usr/sbin/crond -n
root       2927  0.0  0.0 114708   976 pts/1    R+   21:27   0:00 grep --color=auto cron
在路径里看编辑的文件名是否存在
```
![](http://os9ep64t2.bkt.clouddn.com/17-7-17/84009441.jpg)  
绿色部分表示已经启动
3. crontab -l 列出任务计划
```
[root@cs701 ~]# crontab -l
1 10 * 2 * /usr/bin/find /tmp/ -type -f -mtime +100
```
4. 查看某用户的任务计划
```
[root@cs701 ~]#cat /var/spool/cron/root
1 10 * 2 * /usr/bin/find /tmp/ -type -f -mtime +100
[root@cs701 ~]# crontab -r 删除计划
[root@cs701 ~]# crontab -l
no crontab for root

```
### 2.chkconfig系统服务管理
centos7之后不用此命令，了解为主
```
[root@cs701 ~]#chkconfig --list

注意：该输出结果只显示 SysV 服务，并不包含原生 systemd 服务。SysV 配置数据可能被原生 systemd 配置覆盖。 
      如果您想列出 systemd 服务,请执行 'systemctl list-unit-files'。
      欲查看对特定 target 启用的服务请执行
      'systemctl list-dependencies [target]'。

netconsole     	0:关	1:关	2:关	3:关	4:关	5:关	6:关
network        	0:关	1:关	2:开	3:开	4:开	5:开	6:关
```
这2个服务在下面的目录,如果要添加服务，先把服务文件放在此目录下，再开启
```
[root@cs701 ~]#ls /etc/init.d/
functions  netconsole  network  README
```
其他命令：
```
 chkconfig --level 3 network off/on #启动/关闭3级别的服务
 chkconfig --level 345 network off #关闭345级别的服务
 chkconfig --del network #删除服务
 chkconfig --add network #增加服务
 ```

 ```
chrony-dnssrv@.timer                    rhel-domainname.service
```
**系统所有的unit分为以下类型：**   
service 系统服务   
 target 多个unit组成的组  
 device 硬件设备  
 mount 文件系统挂载点  
 automount 自动挂载点  
 path 文件或路径  
 scope 不是由systemd启动的外部进程  
 slice 进程组  
 snapshot systemd快照  
 socket 进程间通信套接字  
 swap  swap文件  
 timer 定时器
 
**unit相关命令：**
```
systemctl list-units # 列出正在运行的unit
systemctl list-units --all #列出所有，包括失败的或者inactive的
systemctl list-units --all --state=inactive   #列出inactive的unit
systemctl list-units --type=service   #列出状态为active的service
systemct is-active crond.service  #查看某个服务是否为active
```
### 5.target
三者关系：
>一个service属于一种类型的unit  
 多个unit组成了一个target  
 一个target里面包含了多个service
 
**target相关命令**
```
systemctl list-unit-files --type=target #查看target文件
systemctl list-dependencies multi-user.target #查看指定target下面有哪些unit
systemctl get-default #查看系统默认的target
systemctl set-default multi-user.target #设置默认的target
```
